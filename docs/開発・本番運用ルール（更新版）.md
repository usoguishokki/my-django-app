# 開発・本番運用ルール（完成版）

## 1. 概要

開発環境と本番環境を分離し、安全に機能追加・修正を行う。
現在は **`settings.py` 一本運用**。

---

## 2. 環境構成

### 開発環境

* パス: `E:\repos\myproject`
* 仮想環境: `venv`

### 本番環境

* パス: `C:\inetpub\wwwroot\sitefolder\myproject`
* 仮想環境: `django_iis_env`

---

## 3. settings運用

### 方針

* `settings.py` 一本
* `myproject.settings` を参照

### 禁止

* `settings.py` と `settings/` の共存

---

## 4. Git運用

### 開発フロー

```text
開発 → 動作確認 → commit → push
```

### 本番反映

```text
本番で git pull
```

---

## 5. staticfiles運用

### ルール

* staticfiles は Git管理しない
* 本番で collectstatic を実行する

```powershell
python manage.py collectstatic --noinput
```

---

## 6. 開発 → 本番手順

```text
① 開発で修正
② 動作確認
③ git commit
④ git push
⑤ 本番で pull
⑥ check
⑦ migrate
⑧ collectstatic
⑨ iisreset
```

---

## 7. 本番反映前チェック

```powershell
git rev-parse HEAD
git status
git branch --show-current
```

### 確認ポイント

* 現在のコミットIDを把握
* 未コミットの変更がないか確認
* 正しいブランチ（main）であること

---

## 8. 本番反映

```powershell
cd C:\inetpub\wwwroot\sitefolder\myproject

# 仮想環境有効化
.\django_iis_env\Scripts\activate

# CSSのローカル差分を削除（pull失敗防止）
git restore myapp/static/css

# 最新コード取得
git pull origin main

# Djangoチェック
python manage.py check

# DB反映
python manage.py migrate

# 静的ファイル反映
python manage.py collectstatic --noinput

# IIS再起動
iisreset
```

### 補足

* `git restore` を入れないと pull が失敗する可能性あり
* CSSは生成物のため本番では保持しない

---

## 9. ロールバック

```powershell
git reset --hard <commitID>
python manage.py collectstatic --noinput
iisreset
```

### ⚠ 注意

* ローカルの変更はすべて削除される
* 実行前に必ず `git status` を確認する
* 本番で実行する場合は影響範囲を確認する

---

## 10. 退避（バックアップ）

作業途中の状態を安全に保存する。

```powershell
git switch -c backup/current-dev-state
git add .
git commit -m "退避ポイント"
git push origin backup/current-dev-state
```

### 確認

```powershell
git branch
git branch -a
```

### ポイント

* `main` に影響しない
* 本番に影響しない
* 作業途中の安全な保存場所

---

## 11. 退避から復元

### ブランチ復元

```powershell
git switch backup/current-dev-state
```

### 状態確認

```powershell
git status
git log --oneline -n 5
```

### 任意の状態へ戻る

```powershell
git log --oneline -n 10
git reset --hard <commitID>
```

---

## 12. 重要ルール

### 禁止

* 本番でコード直接修正
* CSS直接編集

### 必須

```powershell
python manage.py check
python manage.py migrate
python manage.py collectstatic --noinput
```

---

```text
③ web.config確認

```

* backup → 作業退避用
* SCSSが正、CSSは生成物
